# Security Policy for SignBridge AI

We take the security of SignBridge AI seriously. If you believe you have found a security vulnerability in this project, please report it to us immediately using the processes outlined below.

---

## 🛡️ Supported Versions

Only the latest release version is supported with security updates.

| Version | Supported |
| :--- | :--- |
| >= 1.0.0 | Yes |
| < 1.0.0 | No |

---

## 📞 Reporting a Vulnerability

Please do **NOT** open a public issue for security-sensitive bugs or vulnerabilities. Instead, report them privately using one of the following methods:

1. **Email**: Send an encrypted email to `security@example.com` containing:
   - A detailed description of the vulnerability.
   - Step-by-step instructions (or a Proof of Concept script) to reproduce the issue.
   - The potential impact (e.g. privilege escalation, remote code execution).
2. **Encrypted Communications**: Use our PGP Key (ID: `0x7F9B1E3A`) to encrypt sensitive attachments.

---

## ⏱️ Response and Disclosure Timeline

- **Acknowledgement**: You will receive an automated receipt acknowledgement within **24 hours**, and a human response confirming or refuting the vulnerability within **72 hours**.
- **Remediation**: We aim to resolve all high-priority security vulnerabilities within **14 days** of confirmation.
- **Disclosure**: We follow coordinated vulnerability disclosure practices. We ask that you do not disclose the vulnerability publicly until a patch has been merged and a new release tag is generated.
- **Credit**: If your report leads to a security patch, you will be credited in our [CHANGELOG.md](file:///CHANGELOG.md) and security advisories (unless you request anonymity).
